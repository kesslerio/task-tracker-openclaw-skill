# v0.3.1 Summarizer Security Contract

The standup summarizer is the only LLM step in the v0.3.1 morning standup path.
It turns harvested GitHub activity evidence into outcome-oriented draft bullets.
It never records accomplishments and never mutates the board or ledger.

## Transport

- One direct HTTP POST to the local Ollama OpenAI-compatible endpoint.
- Default endpoint: `http://127.0.0.1:11434/v1/chat/completions`.
- Default model id: `qwen3-coder-next:cloud`.
- No API key is configured or sent by task-tracker code; local Ollama owns any
  upstream cloud authentication.
- The request uses `temperature: 0`, `max_tokens`, and a hard timeout.
- The implementation uses Python stdlib `urllib.request`.
- The call has one injectable seam so tests mock success, timeout, HTTP error,
  and malformed responses without live network access.
- There is no tool calling, no function calling, no session history, and no
  fallback to another model.

Runtime knobs:

- `STANDUP_SUMMARIZER_ENABLED`, default enabled.
- `STANDUP_SUMMARIZER_MODEL`, default `qwen3-coder-next:cloud`.
- `STANDUP_SUMMARIZER_BASE_URL`, default `http://127.0.0.1:11434/v1/chat/completions`.
- `STANDUP_SUMMARIZER_TIMEOUT_SECONDS`, default `15`.
- `STANDUP_SUMMARIZER_MAX_TOKENS`, default `600`.

If the service is unavailable, not signed in, slow, returns non-200, or returns
invalid JSON, the standup still succeeds and uses deterministic fallback bullets.

## Input Boundary

The caller extracts a minimal GitHub-only input list from fresh standup evidence:

```json
[
  {"evidence_id": "sha256:github:...", "title": "Clean commit or PR title"}
]
```

The summarizer never receives email, calendar, SMS, phone numbers, raw bodies,
provider state, digests, or adapter-specific metadata. Commit text is JSON in the
HTTP body only; it is never interpolated into a shell command.

## Output Boundary

The model is expected to return JSON content equivalent to:

```json
[
  {"evidence_id": "sha256:github:...", "area": "eng", "bullet": "Outcome text"}
]
```

Validation is hard:

- `evidence_id` must be one of the input IDs. Unknown IDs are rejected.
- `area` is restricted to `eng`, `sales-ops`, `marketing`, or `unclassified`;
  any other value maps to `unclassified`.
- Bullets are collapsed to one line, markdown and mentions are stripped, control
  characters are removed, and length is capped.
- Model text is treated only as untrusted display text.
- Malformed output, non-list output, or output with no valid evidence IDs is a
  failure and falls back deterministically.

## Deterministic Fallback

Fallback never invents work. It groups the same GitHub evidence by a small local
taxonomy and uses the sanitized evidence title as the bullet. The result records:

- `translated: false`.
- Exact configured model id.
- `prompt_version`.
- A disclosure that translation was unavailable.

Fallback is used for disabled mode, timeout, HTTP error, non-200, malformed JSON,
validation failure, or any unexpected summarizer exception.

## Cache

Successful translated results are cached under the Chief-of-Staff state
directory using the existing atomic-write helper and a sidecar flock. The key is:

```text
sha256(canonical_input + prompt_version + exact_model_id)
```

Changing the prompt version or exact model id causes a cache miss and a fresh
one-shot call. A cache hit returns the cached draft without contacting the model.

## Confirm Gate

Summarizer bullets are attached to the harvest result as a draft summary and shown
in the standup as `Draft summary (unconfirmed)`. They are read-only:

- They are not written to `output['completed']`.
- They never set `kind="accomplishment"`.
- They never approve, complete, reconcile, or record work.
- A human confirmation step remains required downstream.
