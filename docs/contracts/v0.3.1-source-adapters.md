# v0.3.1 Source Adapter Contract

U3 adds two thin read-only standup evidence sources: Google Calendar via `gog`
and Dialpad SMS via the existing local SQLite store. Both emit U1 evidence
candidates only. They never mark work DONE, never seed the plan, and never write
provider state.

## Calendar

The calendar adapter reads `STANDUP_CALENDARS` from the environment. Each entry
provides the `gog` command, `calendar_id`, and `account`; optional
`token_env`/`access_token_env` names may pass an access token from the
environment. The adapter does not hardcode or read `~/.config` paths.

Calendar reads call `gog calendar list <calendar_id> --account <account> --from
<evidence_start> --to <evidence_end> --json`. The window is the U1 Pacific
standup evidence window, not wall-clock `--today`.

Calendar semantics:

- Timed past events where the owner accepted or organized the event emit
  `kind="activity"`.
- Timed upcoming accepted or organized events emit `kind="commitment"`.
- Declined events, cancelled events, birthdays, and all-day events are excluded.
- `provider_id` is the event occurrence id. Recurring occurrences must keep
  distinct ids.
- `provider_state` includes event status, self RSVP/organized status, and an
  update token when present.
- `occurred_at` is the event start converted to the local Pacific timezone.

The adapter currently parses the standard Google Calendar event fields:
`status`, `start.dateTime`, `attendees[].self`, `attendees[].email`,
`attendees[].responseStatus`, `organizer.self`, `organizer.email`,
`creator.self`, and `creator.email`. The exact live `gog` JSON field set still
needs operator verification against the ShapeScale account before depending on
live unattended output.

## Dialpad SMS

The Dialpad adapter is SMS-only. Calls, call metadata, and transcripts are out
of v0.3.1 and tracked separately as
`kesslerio/dialpad-openclaw-skill#113`.

The adapter opens `DIALPAD_SMS_DB` in SQLite read-only URI mode
(`mode=ro`). The default path is `/home/art/clawd/logs/sms.db`. It never creates
tables, migrates schema, writes rows, or calls the Dialpad API.

Expected table:

- `messages.dialpad_id` integer unique id.
- `messages.contact_number` text thread key.
- `messages.direction` as `inbound` or `outbound`.
- `messages.timestamp` integer epoch seconds or milliseconds.
- `messages.text` raw SMS body, used only inside the adapter.

The adapter selects messages whose timestamp overlaps the U1 evidence window and
groups them by `contact_number`. A thread is substantive when deterministic
outbound activity crosses either threshold:

- `DIALPAD_SMS_PUSHBACK_MESSAGE_THRESHOLD`, default `3` outbound non-boilerplate
  messages.
- `DIALPAD_SMS_PUSHBACK_CHAR_THRESHOLD`, default `200` outbound non-boilerplate
  characters.

Single-token outbound messages and known automated/boilerplate markers such as
auto-replies, out-of-office text, do-not-reply text, and unsubscribe text do not
count toward those thresholds. There is no LLM or semantic content judgment in
the filter.

One substantive thread emits one `kind="activity"` candidate:

- `provider_id`: `sms:{contact_number}:{evidence_date}`.
- `provider_state`: outbound count, outbound character count, and a deterministic
  digest of the thread.
- `occurred_at`: latest substantive outbound message timestamp in Pacific.
- `title`/`match_title`: deterministic metadata only, for example `SMS thread
  with -4242424242 (3 sent)`.

## Threat Model

Calendar and SMS are evidence candidates, not accomplishments. They cannot
auto-DONE tasks and cannot enter the active plan without a later confirm-gate
unit.

Calendar risks are stale auth, incomplete RSVP fields, declined meetings being
misread as attended, and wall-clock date drift. The adapter mitigates these by
degrading per source on `gog` failure, requiring accepted/organized status,
excluding declined/cancelled/all-day events, and querying the U1 Pacific window.

SMS risks are privacy leakage, accidental semantic judgment, and writes to the
local message store. The adapter mitigates these by keeping raw SMS bodies inside
the adapter only, emitting metadata-only titles, using deterministic thresholds,
opening SQLite read-only, and never invoking the Dialpad API. Raw SMS body text
must not appear in evidence records, standup output, logs, or any field that can
reach a later summarizer.
