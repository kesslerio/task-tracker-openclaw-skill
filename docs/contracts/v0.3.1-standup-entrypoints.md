# v0.3.1 Standup Entrypoint Contract

Typed `/standup` and the 8am scheduled standup are one deterministic path:

```text
telegram-commands.sh daily -> standup.py -> generate_standup
```

There is no cron-only implementation, no `--cron` mode, and no alternate
summarizer or harvest path for the scheduled run. The shell wrapper is only the
Telegram/OpenClaw command boundary; it delegates to `standup.py` and may forward
ordinary `standup.py` inspection flags such as `--compact-json` for tests and
operators.

## Cron Descriptor

`standup.standup_cron_descriptor()` is a code-only template. It returns a
deterministic command cron:

- `payload.kind == "command"`.
- `payload.argv` changes into the installed scripts directory and runs
  `bash telegram-commands.sh daily`.
- `delivery.mode == "announce"`.
- `delivery.chat_id_env == "TELEGRAM_CHAT_ID_PRODUCTIVITY"`.
- `delivery.topic_env == "OPENCLAW_TOPIC_PRODUCTIVITY_STANDUP"`.

The descriptor embeds environment variable names only. It must not commit a real
Telegram chat id, topic id value, token, live cron registration, or
`openclaw.json` state. Registering this descriptor, swapping the live 8am cron,
and deleting the legacy standup cron are deferred operator steps.

The descriptor uses `sh -c`, not a login shell. A login shell can reload host
profile state and drift away from the environment supplied to the command cron;
the parity contract requires the typed command and cron command to inherit the
same effective runtime environment.

## Deterministic Work

Both entrypoints run the same `standup.py` main function and therefore the same
U2-U7 deterministic sequence:

- resolve the U1 standup window;
- run the standup evidence harvest;
- record per-source health receipts under the `standup` ritual;
- reconcile user-stated completions with evidence candidates;
- attach the U4 pinned summarizer draft only through the existing summarizer
  boundary;
- render the requested markdown or JSON output shape.

The standup path remains read-only for evidence: harvested records and model
drafts do not mark tasks done, mutate the board, or register accomplishments.

## Guard

`tests/test_standup.py` contains the U8 parity characterization. For a fixed
fixture it runs both real shell entrypoints:

```text
bash scripts/telegram-commands.sh daily --compact-json ...
sh -c 'cd scripts && bash telegram-commands.sh daily --compact-json ...'
```

The test asserts byte-identical compact JSON, an idempotent rerun, source health
receipts for the harvest, disabled/mocked LLM behavior, and code-only descriptor
hygiene. Any future cron-only branch, login-shell environment drift, committed
chat id, or output-shape divergence should fail this guard before deployment.
